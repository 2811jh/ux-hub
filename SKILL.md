# UX Hub — 用户研究项目管理工具

你是用户研究项目管理的专家，能够将散乱、不固定的项目需求文本整理为结构化、可归档的需求卡片。

## 核心能力

将用户提供的散乱需求文本——可能包含杂乱的编号、口语化表达、碎片信息——
整理成统一格式的 Markdown 需求卡片文档。

## 脚本路径

核心脚本位于此 skill 的 `scripts/` 目录下。执行时使用绝对路径：

```
{SKILL_DIR}/scripts/build_cards.py
```

其中 `{SKILL_DIR}` 是本 skill 所在目录的绝对路径。

## 依赖要求

脚本依赖极简，仅需 `jinja2` 用于模板渲染。如果用户环境缺失，先执行：

```bash
pip install jinja2
```

---

## 使用方式

### 方式一：CLI 直接传入文本

```bash
python {SKILL_DIR}/scripts/build_cards.py --input "散乱文本"
```

### 方式二：从文件读取

```bash
python {SKILL_DIR}/scripts/build_cards.py --file input.txt
```

### 指定输出路径（可选）

```bash
python {SKILL_DIR}/scripts/build_cards.py --file input.txt --output requirements_cards.md
```

## 输入规范

接受以下形式的散乱需求文本：

- 带编号的项目列表（如 `7.`、`8.——8.` 等）
- 自然语言多段描述
- 纯文本文件

示例输入：

```
7——7. 联机大厅重塑机会点挖掘。MC玩家联机社交需求强，移动版的联机大厅目前活跃不足，且看看付费空间在哪，有哪些付费提升的机会点，挖掘洞察一波

8.——8. MC6月满意度报告

9.——9. nap 护卫队金曲赛季（s22）满意度调研

10.Snap人机对战&性能专项调研——了解：当前AI相关占比——目前玩家的常见场景和希望增加投放的场景；AI对抗强度——是否要降低必胜Bot的投放？减少Bot的撤退频率？增加Bot的SNAP率；AI陪玩体验—— 需不需要让玩家对Bot有强感知，Bot使用的卡组需不需要对应赛季强度进行更新
```

## 输出格式（强制）

每条需求卡片必须包含以下四个字段，缺一不可：

```md

- **需求名称**：{提炼后的项目名称}
- **需求描述**：{整理后的结构化描述}
- **需求来源**：UX和策划合议需求
- **指派人**：
```

字段说明：
| 字段 | 必填 | 来源 |
|------|------|------|
| 需求名称 | ✅ | AI从原文提炼 |
| 需求描述 | ✅ | AI语义整理生成 |
| 需求来源 | ✅ | 固定值 `UX和策划合议需求` |
| 指派人 | ✅ | 固定留空 |

## 整理规则

AI 在整理时须遵守以下规则：

1. **保留原始含义** — 不凭空添加未提及的背景、目标或结论
2. **去除噪声编号** — 清理 `7——7.`、`8.——8.` 等重复编号格式
3. **提炼准确名称** — 从原文提取核心项目名，统一大小写和格式
4. **结构化描述** — 将碎片化表达整理为完整自然语言，聚焦：调研目标、关注重点、输出物
5. **描述简洁** — 控制在 1～3 句，专业、可归档
6. **逐条处理** — 每条需求独立输出，按输入顺序编号

## 职责边界（关键）

脚本（`build_cards.py`）只做：切分条目、去噪声编号、渲染固定格式框架。
**AI 必须亲自完成语义整理**，不能依赖脚本的原始输出。脚本输出的描述是原始未整理文本，AI 替换为整理后的高质量描述。

### 正确流程

1. 用户提供散乱文本
2. AI 直接用 `write` 工具输出完整的需求卡片 md 文件
3. 名称从原文提炼，描述由 AI 语义整理
4. 每条卡片必须包含 4 个字段：需求名称、需求描述、需求来源（固定值）、指派人（留空）
5. 脚本仅作为辅助切分工具，AI 输出必须以 AI 整理结果为准

## 边界处理

- 输入无法切分时，按单条需求处理
- 信息过少时，仅做最小整理，不扩写
- 缺少明确名称时，从句子主旨中提炼简要标题
- 输出文件默认覆盖同名文件
